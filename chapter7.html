Гибридный Интранет

Автономная радиосеть не обязана быть абсолютно изолированной. Истинная мощь архитектуры раскрывается, когда мы создаем мосты (Bridges) между медленным, но независимым эфиром LoRa и высокоскоростными локальными сетями (LAN), сохраняя полный суверенитет над собственными данными.

7.1. Архитектура Шлюза (Gateway)

Для объединения двух разных физических сред используется Шлюз. В нашем случае это связка радиомодуля (nRF52840) и микрокомпьютера.

Радиомодуль подключается к SBC через Serial-порт (USB). Радио принимает зашифрованные пакеты из эфира, передает их по кабелю в Linux-систему, где специальный скрипт обрабатывает их и отправляет в локальную сеть.

Это позволяет, например, двум автономным Mesh-сетям в разных городах общаться друг с другом через ваш собственный VPN-тоннель, минуя серверы публичных операторов.

7.2. Протокол MQTT

Для маршрутизации внутри локальной сети идеально подходит протокол MQTT. На Raspberry Pi разворачивается локальный сервер (Брокер) Mosquitto.

Каждое текстовое сообщение или GPS-координата из радиоэфира публикуется в определенный Топик. Любое устройство в вашей домашней сети (ПК, смартфон с Wi-Fi, сервер умного дома Home Assistant) может подписаться на этот топик и мгновенно получать данные из радиоэфира, не имея собственной антенны.

7.3. Федерация Matrix

Если MQTT — это машинный язык, то для удобного общения людей мы используем протокол Matrix.

Архитектура федерации означает, что не существует единого центрального сервера. Вы поднимаете свой Homeserver (например, Synapse) на домашнем мини-ПК.

Python-мост: Пишется небольшой скрипт-демон. Он слушает локальный MQTT-брокер. Как только с улицы по радио прилетает пакет от человека с рацией, скрипт перехватывает его и отправляет в чат Matrix. Пользователь, сидящий за ПК дома в удобном клиенте (Element), видит это сообщение и может ответить на него текстом, который пройдет обратный путь: ПК -> Matrix -> Python -> MQTT -> USB -> LoRa -> Эфир.
